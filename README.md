# Un Poco de Tango — Ruleta

운뽀꼬데탱고에서 **Leader / Follower / Orchestra** 세 그룹의 참가자 중 무작위로 한 명씩을 뽑아 카드 플립 애니메이션으로 공개하는 단일 페이지 웹앱입니다. 빌드 도구나 외부 라이브러리 없이 바닐라 HTML / CSS / ES6 모듈만 사용하며, GitHub Pages 정적 호스팅에 맞춰져 있습니다.

🔗 **Live Demo** — https://myeongheum.github.io/un-poco-ruleta/

## 사용 방법

### 1. 참가자 추가하기

각 컬럼 상단의 입력란에 이름을 적고 `+` 버튼을 누르거나 **Enter** 키를 누릅니다.

- 앞뒤 공백은 자동으로 제거되며, 빈 이름은 등록되지 않습니다.
- 동일 컬럼에 이미 같은 이름이 있으면(대소문자·중간 공백 차이 무시) 입력란이 살짝 흔들리며 등록을 거부합니다. 스크린리더 사용자에게는 음성 안내가 출력됩니다.

### 2. 참가자 삭제하기

목록에 표시된 각 항목 오른쪽의 `✕` 버튼을 누르면 그 사람만 삭제됩니다.

### 3. 자동 제거(Auto-remove) 설정

각 컬럼 헤더 우측의 체크박스가 자동 제거 토글입니다.

- **ON** — 추첨에서 뽑힌 사람이 모달을 닫는 순간 해당 컬럼에서 자동 삭제됩니다.
- **OFF** — 뽑힌 사람도 그대로 남아 다음 추첨에 다시 후보가 됩니다.

| 컬럼 | 기본값 | 의도 |
|---|---|---|
| Leaders | **ON** | 한 번 뽑힌 리더는 보통 다시 뽑지 않음 |
| Followers | **ON** | 팔로워도 마찬가지 |
| Orchestras | **OFF** | 같은 악단이 연달아 등장하는 것이 자연스러움 |

### 4. 추첨 실행 — `Start`

푸터의 `Start` 버튼을 누르면 결과 모달이 열립니다.

- 세 컬럼 모두 한 명 이상이어야 추첨이 실행됩니다. 하나라도 비어 있으면 버튼을 눌러도 아무 일이 일어나지 않습니다.

### 5. 카드 공개와 결과 확인

모달 안에는 뒷면이 보이는 세 장의 카드(`Who is Next?`)가 가로로 배치됩니다.

1. 카드를 **클릭**하면 한 장씩 뒤집힙니다. 플립 애니메이션은 약 2초 동안 탄성 있게 흔들리다 앞면을 드러냅니다.
2. 카드 앞면에는 **역할**, **이름**, **카드 심볼**이 표시됩니다.
3. 세 장이 모두 공개되면 잠시 후 `Back` 버튼이 나타납니다.
4. `Back` 버튼을 누르거나 `Esc` 키를 누르면 모달이 닫힙니다.

### 6. 데이터 내보내기 / 불러오기

푸터의 `내보내기` / `불러오기` 버튼으로 참가자 명단과 자동 제거 설정을 파일로 주고받을 수 있습니다.

- `내보내기` — 현재 상태를 `AppSettings.json` 파일로 다운로드합니다.
- `불러오기` — 같은 형식의 JSON 파일을 선택하면 상태가 즉시 복원됩니다. 파일이 깨졌거나 형식이 잘못되었다면 안전한 기본값으로 보정되며, 파싱 자체가 실패하면 알림이 뜨고 기존 상태는 그대로 유지됩니다.

> 모든 상태는 브라우저의 `localStorage`(키: `unpoco-ruleta-settings`)에도 자동 저장되므로, 새로고침하거나 탭을 다시 열어도 그대로 남아 있습니다.

`AppSettings.json` 예시:

```json
{
  "leaders": ["Alice", "Carla"],
  "followers": ["Bob", "Diego"],
  "orchestras": ["Trio Tango", "Sexteto Sol"],
  "isAutoRemoveLeaders": true,
  "isAutoRemoveFollowers": true,
  "isAutoRemoveOrchestras": false
}
```

## 추첨 방식

### 균등 분포 무작위 추출

`Math.random()` 기반의 단순한 균등 분포 추출입니다. 가중치나 사전 순번은 없습니다.

```js
function pickRandom(arr) {
  return arr[Math.floor(Math.random() * arr.length)];
}
```

이 함수가 세 컬렉션(`leaders`, `followers`, `orchestras`)에 각각 독립적으로 적용되어, **한 번의 `Start`로 정확히 3명**(역할별 1명씩)이 선출됩니다.

### 빈 컬렉션 가드

세 그룹 중 **한 그룹이라도 비어 있으면 추첨이 실행되지 않습니다**. 모달도 열리지 않고 조용히 무시됩니다. 행사 전 모든 컬럼에 최소 1명 이상이 들어 있는지 확인하세요.

### 자동 제거 동작 순서

1. 세 명을 추첨 → 모달 표시
2. 카드 한 장씩 클릭 → 플립
3. 모달을 닫는 시점에 각 역할의 자동 제거 토글을 확인
4. ON인 역할에 한해 당첨자를 컬렉션에서 제거
5. `localStorage` 갱신 후 해당 컬럼만 다시 렌더링

자동 제거가 OFF인 컬럼의 당첨자는 그대로 남아 다음 추첨에서도 후보가 됩니다.

### 카드 심볼 매핑

| 역할 | 심볼 |
|---|---|
| Leader | ♠ |
| Follower | ♥ |
| Orchestra | ♣ |

## 로컬에서 실행하기

스크립트가 ES6 모듈(`<script type="module">`)로 로드되기 때문에 `index.html`을 `file://`로 직접 열면 CORS 오류로 실패합니다. 가벼운 정적 서버 하나만 띄우면 됩니다.

```bash
# 택 1
python -m http.server 8000
npx http-server -p 8000
```

이후 브라우저에서 `http://localhost:8000` 으로 접속합니다. 빌드 단계는 없으며, 코드를 수정한 뒤 새로고침이면 끝입니다.

## 파일 구조 (요약)

| 경로 | 책임 |
|---|---|
| `index.html` | 단일 페이지 마크업, 카드/참가자 `<template>` 정의, 결과 모달 골격 |
| `scripts/app.js` | 진입점. 이벤트 바인딩과 사용자 인터랙션 흐름 조립 |
| `scripts/store.js` | 상태 스키마·정규화·`localStorage` 영속화 |
| `scripts/render.js` | 참가자 목록·자동 제거 체크박스 DOM 렌더링 |
| `scripts/draw.js` | 추첨 로직 (`executeDraw`, `pickRandom`) |
| `scripts/dialog.js` | 결과 모달과 카드 플립 시퀀스 |
| `scripts/io.js` | JSON 내보내기/불러오기 |
| `styles/theme.css` | 골드 팔레트, 폰트, 그림자 등 CSS 변수 |
| `styles/main.css` | 3컬럼 레이아웃, 헤더·푸터, 참가자 리스트 |
| `styles/card.css` | 카드 플립 키프레임 (2초, 탄성 이징) |
| `styles/dialog.css` | 결과 모달 오버레이와 카드 그리드 |

## 배포 (GitHub Pages)

별도 워크플로 없이 GitHub Pages로 즉시 배포할 수 있습니다.

1. 레포 `Settings → Pages` 진입
2. **Source**를 `main` 브랜치의 `/ (root)`로 지정
3. 저장하면 잠시 후 `https://myeongheum.github.io/un-poco-ruleta/` 로 배포 완료

모든 경로가 상대 경로이므로 base path 설정은 필요 없습니다.
